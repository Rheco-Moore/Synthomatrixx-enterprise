# Design Principles

## Cloud First

Cloud services are preferred over on-premises infrastructure whenever practical.

---

## Identity First

Identity is the primary security boundary.

Every user authenticates using Microsoft Entra ID.

---

## Zero Trust

No user or device is automatically trusted.

Authentication and authorization are continuously evaluated using:

- MFA
- Conditional Access
- Device Compliance
- Identity Risk

---

## Least Privilege

Users receive only the permissions required to perform their job.

---

## Standardisation

Devices, applications and services follow standardized deployment procedures.

---

## Automation

Manual administrative tasks should be automated whenever possible.

---

## Monitoring

Security and operational telemetry is collected continuously.

---

## Documentation

Every production configuration must be documented.
